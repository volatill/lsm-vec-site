Why LSM-Vec?

A vector database designed to be cheap to run — and priced that way.

Predictably cheap to run

A disk-oriented design keeps the memory footprint small and flat as your data grows. That makes the service inexpensive to host — and we pass the savings on as flat monthly pricing, with no per-query meter.

Performance close to in-memory

Most of the HNSW index lives in a graph-oriented LSM-tree on disk (Aster), with a page cache and batched reads on the hot path. Search and update stay fast — see the benchmarks.

Managed, or run it yourself

Start on the hosted service over HTTPS with an API key. Or self-host the same open-source engine behind the same HTTP API — a clear migration path to self-hosting, with no lock-in.

From zero to first query in minutes

Install the client, point it at your endpoint, insert your vectors, and search. You bring the embeddings; we handle storage, k-NN, and metadata filtering.

Full quickstart → 
# pip install lsmvec-client
from lsmvec_client import Client

client = Client(api_key="sk-live-...",
                base_url="https://api.lsmvec.com")

client.insert(1, [0.10, 0.20, ...],
              metadata={"category": "docs"})

hits = client.search([0.10, 0.20, ...], k=10)
for h in hits:
    print(h.id, h.distance)

Features

Everything you need for production similarity search.

Filtered similarity search

Combine vector k-NN with rich metadata predicates ($eq, $in, ranges, $and/$or, and more) in a single query.

Per-vector JSON payloads

Attach a JSON document to every vector — set, replace, or merge-patch it independently of the vector.

Fast bulk build

Load a new index in one pass for a faster initial import than per-vector inserts.

Delete & upsert

Insert, replace, and delete vectors with consistent search results under continuous writes.

L2 & cosine, SIMD-accelerated

AVX2/SSE2-accelerated distance for Euclidean and cosine similarity.

Python client + HTTP API

A dependency-free Python client, plus a plain REST API you can call from any language.
